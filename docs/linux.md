# Linux 

## Linux 
* **Linux** - это популярная бесплатная операционная система с открытым исходным кодом.

#### Компоненты Linux 
* **Загрузчик** - это низкоуровневое программное обеспечение выполняет первичную инициализацию и проверку железа компьютера и передаёт управление загрузчику операционной системы. <br>
* **Ядро** - это центральная часть операционной системы, программный код и функции управления аппаратным обеспечением и пользовательскими процессами. Ядро предоставляет пользовательским процессам сервисы и интерфейсы для взаимодействия с аппаратным обеспечением и окружающим миром с помощью механизма системных вызовов. 
>Современное ядро Linux является монолитным с динамически подгружаемыми модулями. 
>Это означает, что само по себе ядро включает минимальные необходимые компоненты, а его функционал может расширяться с помощью модулей, причём для подключения модулей нет необходимости в перекомпиляции ядра. 
* **Система инициализации** - это первый процесс с идентификатором 1, на плечи которого ложатся задачи по созданию пользовательского окружения: монтирование файловых систем, запуск сервисов демонов и процессов. 
>  `systemd`
* **Демоны** - это фоновые процессы, которые работают независимо от пользовательского сеанса и обычно запускаются при старте системы. Они не имеют управления с терминала и выполняют различные сервисные функции. <br>
!!! Примеры демново

    `sshd` — обслуживание SSH-подключений

    `cron` — выполнение задач по расписанию

    `nginx/httpd` — веб-серверы

    `mysqld/postgresql` — базы данных

    `systemd-journald` — системное логирование

    `networkd` — управление сетью
* **Командная оболочка** - Оболочка и консольные утилиты появились во времена физических терминалов — устройств ввода-вывода с экраном и клавиатурой.
* **Библиотеки и утилиты**. Наборы переиспользуемых функций для приложений. 
!!! Пример 
    Файловые операции: `ls`, `cd`, `cp`, `mv`, `rm`, `mkdir`, `touch`

    Текстовая обработка: `grep`, `sed`, `awk`, `cut`, `sort`, `uniq`

    Управление процессами: `ps`, `top`, `htop`, `kill`, `jobs`

    Сеть: `ping`, `curl`, `wget`, `ssh`, `scp`, `netstat`
* **Графический сервер** -  это программа, которая управляет графическим выводом на экран и обрабатывает ввод от устройств (клавиатура, мышь, тачпад). Он предоставляет интерфейс для графических приложений и отрисовывает их окна.<br>
* **Пакетный менеджер** - это фундамент современных дистрибутивов. Ядро, загрузчик или утилиты для работы с файловой системой начинают свою жизнь с исходного кода. <br>
!!! Дополнение
    Пакетный менеджер в Linux

    Это программа, предназначенная для установки, удаления и управления пакетами. Пакетный менеджер позволяет удобно устанавливать и обновлять программы, а также обеспечивает автоматическое разрешение зависимостей между пакетами. Пакетные менеджеры ведут базу данных изменений и каждая установка или удаление пакета создаёт в такой базе данных транзакцию. Транзакция позволяет откатить изменения, если что-то пошло не так. На этом свойстве пакетных менеджеров основаны системы управления конфигурацией, такие как Ansible, Chef и Puppet.


Итоговая схема взаимодействия: 
```
Загрузчик → Ядро → init (systemd) → Демоны + Графический сервер → Оболочка (Shell)
                ↓
         Пакетный менеджер ↔ Библиотеки и утилиты
                ↓
         Пользовательские приложения
```

#### Группы процессов, задачи и сессии
* Процессы - это экземпляр выполняющейся программы, имеющий свой идентификатор (PID), адресное пространство и контекст выполнения. <br>
**Каждый процесс имеет**
    - PID (Process ID) — уникальный идентификатор
    - PPID (Parent Process ID) — идентификатор родителя
    - PGID (Process Group ID) — идентификатор группы процессов
    - SID (Session ID) — идентификатор сессии
    - UID/GID — владелец процесса
    - Адресное пространство
    - Стек, куча, сегменты кода и данных
    - Таблица файловых дескрипторов
* Задача (Job) — это процесс или группа процессов, запущенных из командной оболочки (shell). Shell управляет задачами как единым целым. <br>
* Сессия — это набор групп процессов, объединенных одним управляющим терминалом. Сессия имеет лидера сессии (session leader) — процесс с SID = PID. <br>
## Command

### ls 
* `ls` - просмотр объектов в каталоге 

| Флаг | Описание | Пример |
|-------|----------|---------|
| `-l`     | Подробный список (права, владелец, размер, дата)     | `ls -l`     |
| `-a`     | Показывает все файлы, даже скрытые       | `ls -a`      |
| `-h`     | Человеко-читаемый размер (KB, MB, GB)   | `ls -lh`      |
| `-r`     | Обратный порядок сортировки   | `ls -hr`      |
| `-r`     | Сортировка по времени изменения (новые первыми)   | `ls -r`      |
| `-S`     | Сортировка по размеру (большие первыми)   | `ls -lS`      |

### rm 
* `rm <file_namr>` - удаление файла, для директории выдаст ошибку

* `rm -f ` - принудительное удаление

### ps
* `ps` - показывает все процессы запущенные в текущей сессии

| Флаг | Описание | Пример |
|-------|----------|---------|
| `aux`     | Все процессы (a=все, u=пользовательский формат, x=демоны)     | `ps aux`     |
| `-ef`     | Стандартный полный список     | `ls -ef`     |
| `-u`     | Процессы конкретного пользователя | `ls -u <user>`     |
| `-p`     | Конкретный процесс по ID | `ps -p <PID>`     |
| `--forest`     | Древовидный вид | `ps aux --forest`     |

---------

### lsof
* `lsof` - открытые файлы и сокеты

> В Linux "всё есть файл" - обычные файлы, директории, сетевые сокеты, pipes.

| Флаг | Описание | Пример |
|-------|----------|---------|
| `-i`     | Сетевые соединения (TCP/UDP)| `lsof -i` или `lsof -i :3000`|
| `-p`     | Файлы открытые процессом | `lsof -p <PID>`     |

---------

### ping
* `ping` - утилита для проверки целостности и качества соединения 
* `ping -c` - количество запросов 

---------

### cURL
* `curl` - утилита для отправки http запросов

!!! Подсказка
    Есть хороший инструмент Postman. В нем каждый запрос можно представить через cURL и после выполнить в терминале. Обратите внимание, что могу добавиться дополнительные заголовки, параметры, метод и тело запроса
